Automount USB drive

For ext4:

  • Edit /etc/fstab
  • Get details with lsblk -f
  • Create the mount point (anywhere, like /home/omk/USB_Drivers/Backup_1)
  • in etc/fstab UUID /mnt/point ext4 defaults,noatime,nofail 0 2

For exFat:

UUID /mnt/point exfat defaults,noatime,nofail,auto,umask=000,users,rw 0 0

Not sure about this, actually

FFMPEG

Combine video + audio

ffmpeg -i video.mp4 -i audio.wav -c:v copy -c:a aac -map 0:v:0 -map 1:a:0 output.mp4

combinar varios arquivos de media em um só no ffmpeg

  • criar files.txt: 
    file 'file1.mp3'
file 'file2.mp3'
etc...
  • Run: 
    ffmpeg -f concat -safe 0 -i files.txt -c copy output.mkv

batch convert WAV to MP3/OGG/FLAC:

  • MP3: for i in *.wav; do ffmpeg -i "$i" -b:a 320000 "${i%.*}.mp3"; done
  • OGG: for i in *.wav; do ffmpeg -i "$i" -b:a 320000 "${i%.*}.ogg"; done
  • FLAC: for i in *.wav; do ffmpeg -i "$i" "${i%.*}.flac"; done
  • All three: for i in *.wav; do ffmpeg -i "$i" -b:a 320000 "${i%.*}.mp3" -b:a 320000 "${i%.*}.ogg" "${i%.*}.flac"; done

FORMATTING DISKS/USB

gparted (Fat32)

lsblk # check which disk
sudo umount /dev/sdX1
sudo parted /dev/sdX --script -- mklabel msdos
sudo parted /dev/sdX --script -- mkpart primary fat32 1MiB 100%
sudo mkfs.vfat -F32 /dev/sdX1
sudo parted /dev/sdX --script print

Slightly different terminal method

  • check location df
  • unmount sudo umount /dev/sdb1
  • format sudo mkfs.exfat /dev/sdb1
  • check if it worked sudo fsck /dev/sdb1

Generating CV (for Jekyll website)

./update_cv_pdf

#!/bin/bash

sed -i '14,$d' cv_pdf/cv_pdf.md ## deletes after line 14

sed -n '25,$p' cv.md > cv_temp ## copies from original from line 25 into temp file

sed -i '/Back to the top/d' cv_temp ## delete lines with 'back to the top'

line=13

sed -i -e "${line}r cv_temp" cv_pdf/cv_pdf.md ## inserts temp file into cv_pdf after line 13

SSH

  • On server machine (remote):
    • ip a to get host address
  • On local machine
    • ssh USER@HOSTADDRESS

systemctl status sshd

sudo systemctl start sshd sudo systemctl enable sshd

Backup and sync

rsync -a --delete --quiet -e ssh /path/to/backup remoteuser@remotehost:/location/of/backup

  • --delete = file deleted on the source will be delete on backup
  • -e ssh = access to remote server
  • Script for the above, at /etc/cron.daily/backup:
#!/bin/sh
rsync -a --delete --quiet -e ssh /path/to/backup remoteuser@remotehost:/location/of/backup

Personal Configs

Mouse:

/etc/X11/xorg.conf.d/10-mouse.conf

Section "InputClass"
	Identifier "My Mouse"
	MatchIsPointer "yes"
# some curved deceleration
	Option "AdaptiveDeceleration" "2"
# linear deceleration (mouse speed reduction)
	Option "ConstantDeceleration" "2"
	Option "NaturalScrolling" "on"
	Option "AccelerationProfile" "-0.5"
	Option "AccelerationScheme" "none"
	Option "AccelSpeed" "-1"
EndSection

Touchpad:

/etc/X11/xorg.conf.d/30-touchpad.conf

Section "InputClass"
    Identifier "touchpad"
    Driver "libinput"
    MatchIsTouchpad "on"
    Option "NaturalScrolling" "on"
    Option "Tapping" "on"
EndSection

Keyboard layout switch:

/etc/X11/xorg.conf.d/00-keyboard.conf

Section "InputClass"
        Identifier "system-keyboard"
        MatchIsKeyboard "on"
        Option "XkbLayout" "us,br"
        Option "XkbModel" "pc105"
        Option "XkbOptions" "grp:alt_shift_toggle"
EndSection

on Polybar:

[module/xkeyboard]

label-indicator-margin = 0
label-indicator-underline = #000044

PDANet WiFi Hotspot

For window managers, you need to set the proxy for each program.

  • Firefox:
    • Settings, all the way down to Network Settings.
    • Manual proxy configuration
      • 192.168.49.1 port 8000
      • check box “Also use ths proxy for HTTPS”
  • Terminal (for updates)
    • on .bashrc: add: 
      export http_proxy=192.168.49.1:8000
export https_proxy=192.168.49.1:8000
  • Insync
    • Settings, Network, Manual proxy

On tutorials, I saw people adding `Defaults env_keep +=”https_proxy” to sudoers file (via command visudo)

Change DNS server

NextDNS systemd-resolved

Use the following in /etc/systemd/resolved.conf:

[Resolve]
DNS=45.90.28.0#dffb84.dns.nextdns.io
DNS=2a07:a8c0::#dffb84.dns.nextdns.io
DNS=45.90.30.0#dffb84.dns.nextdns.io
DNS=2a07:a8c1::#dffb84.dns.nextdns.io
DNSOverTLS=yes

sudo systemctl restart systemd-resolved
